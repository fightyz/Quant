# 集思录投资论坛摘要

从集思录（jisilu.cn）的最新讨论中提取有投资决策价值的信息，生成结构化摘要。

## 时间窗口参数

默认值：
- 帖子筛选：**运行当天及前一天**（即"最近 1 天"覆盖完整的两个自然日，例如今天是 3 月 8 日，则筛选 3 月 7 日和 3 月 8 日有更新的帖子）
- 回复回顾：每个帖子最近 **2 周**内的回复

用户可以在提示中覆盖，例如：
- "帮我看看集思录最近 3 天的讨论" → 帖子筛选改为运行当天及前 3 天
- "回顾最近 1 个月的内容" → 回复回顾改为 1 个月

如果用户指定了特定主题（如"转债"、"原油"），仍然先获取完整帖子列表，然后从标题和摘要中筛选所有相关帖子，不要过早排除——有些帖子标题不含关键词但内容高度相关。

## 工作流程

### 第一步：获取帖子列表并粗筛

使用 WebFetch 抓取 `https://www.jisilu.cn/explore/`，提取所有可见帖子的：
- 标题
- 链接 URL
- 最后回复时间

根据时间窗口筛选后，将帖子分为两档：

**A档（优先深入）**——标题明确涉及以下内容的帖子：
- 市场走势分析、宏观经济判断、风险预警
- 具体投资品种讨论（转债、ETF、期权、商品、个股等）
- 投资策略和实盘记录
- 资金流向、政策变化、套利机会

**B档（快速浏览）**——其余帖子，仅从标题和摘要判断是否值得深入

如果帖子总数超过 20 个，先处理 A 档。B 档仅在时间和上下文允许时补充。

### 第二步：并行抓取帖子详情（含图片）

对 A 档帖子，使用 WebFetch 并行获取每个帖子的内容。提取：
- 发帖人的核心观点
- 最近一段时间的回复内容（按时间窗口参数）
- 回复中的关键讨论和争论
- **所有图片 URL**（通常格式为 `https://www.jisilu.cn/uploads/...`）

集思录的帖子回复可能是动态加载或分页的。如果第一页看不到最新回复，追加 `?page=last` 重新抓取。

#### 图片处理

集思录帖子经常包含数据图表（Excel 截图、K线图、指标表格等），这些图片往往包含比文字更关键的数据。在抓取每个帖子时，如果发现图片 URL，**必须**执行以下步骤：

1. 用 Bash 工具执行 `curl -s -o /tmp/jisilu_<帖子ID>_<序号>.png "<图片URL>"` 下载图片
2. 检查图片尺寸（跨平台）：
   ```bash
   # Linux (ImageMagick)
   magick identify -format "%w %h" /tmp/jisilu_<帖子ID>_<序号>.png
   # macOS (sips)
   # sips -g pixelWidth -g pixelHeight /tmp/jisilu_<帖子ID>_<序号>.png
   ```
3. 如果图片高度 > 600px，先放大再分段裁剪以提高辨识度：
   ```bash
   # Linux (ImageMagick)
   magick /tmp/jisilu_<帖子ID>_<序号>.png -resize 2000x2000 /tmp/jisilu_<帖子ID>_<序号>_scaled.png
   # 按高度 500-600px 分段裁剪，注意相邻段重叠 100px 避免截断内容
   magick /tmp/jisilu_..._scaled.png -crop <width>x600+0+0 /tmp/jisilu_..._part1.png
   magick /tmp/jisilu_..._scaled.png -crop <width>x600+0+500 /tmp/jisilu_..._part2.png

   # macOS (sips) 替代命令：
   # sips -Z 2000 /tmp/jisilu_<帖子ID>_<序号>.png --out /tmp/jisilu_<帖子ID>_<序号>_scaled.png
   # sips -c 600 <width> --cropOffset 0 0 /tmp/jisilu_..._scaled.png --out /tmp/jisilu_..._part1.png
   # sips -c 600 <width> --cropOffset 500 0 /tmp/jisilu_..._scaled.png --out /tmp/jisilu_..._part2.png
   ```
4. 用 Read 工具逐段查看裁剪后的图片，提取数据
5. 如果某些文字辨识不清，在输出中标注"[辨识不清]"而非猜测

跳过图片处理等于丢失关键信息。如果因为权限问题无法下载图片，在输出中明确标注"[图片未能分析]"并附上图片 URL。

### 第三步：生成结构化摘要

按以下框架组织输出。只输出有实质信息的分类，不需要每个都有。

```
## 一、宏观与市场信号
（大盘走势判断、政策变化、国际局势对市场影响等）

## 二、品种与策略
（转债、ETF、个股、商品、期权等具体品种讨论和策略分享）

## 三、资金与渠道
（资金流向、开户、跨境、套利机会等）

## 四、投资者教育/警示
（踩坑经验、风险提示、常见误区等）

## 关键投资决策参考总结
（用表格形式汇总：维度 | 信号 | 建议方向）
```

每个条目必须包含：
- **[原帖链接](URL)**：紧跟在条目标题之后
- **关键数据**：具体数字、百分比、日期，而非模糊描述
- **多方观点**：如果帖子中有争论，呈现主要的正反方观点

### 质量检查清单

输出前逐项确认：
- [ ] 每个条目都附有原帖链接
- [ ] 所有引用的数据都标注了时间（避免将旧数据当新数据呈现）
- [ ] 帖子中发现的图片都已尝试下载和分析
- [ ] 图片中提取的数据已整合到对应条目中
- [ ] 输出使用中文
